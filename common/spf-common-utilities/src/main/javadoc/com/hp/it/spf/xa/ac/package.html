This package contains the common data model for all admission control functions, including the healthcheck Web application, and the AdmissionControlFilter servlet filter. These are portal components which regulate entry into the system based on various criteria (such as system availability), working with the load-balancers (ie in the infrastructure) to provide load-redirection and failover capability beyond what the load-balancers themselves can natively provide.

Note: SPF portal and portlet developers should never need to use this package.

HealthcheckStatus

{@link com.hp.it.spf.xa.ac.HealthcheckStatus} is an abstract base class for all kinds of healthcheck status outcome.

OpenStatus

{@link com.hp.it.spf.xa.ac.OpenStatus} is a concrete HealthcheckStatus meaning that the site is fully operational (ie is both working and open to end-users).

ClosedStatus

{@link com.hp.it.spf.xa.ac.ClosedStatus} is a concrete HealthcheckStatus meaning that the site is internally operational (ie is working) but has been administratively closed to end-users.

DownStatus

{@link com.hp.it.spf.xa.ac.DownStatus} is a concrete HealthcheckStatus meaning that the site is non-operational (ie is not working).
