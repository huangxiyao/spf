1. Overview

Web applications that require the look-and-feel of the HPWeb UI standards can make use of this HPWeb layout provided as an add-on to the WPA framework.  Using the HPWeb layout allows developers to focus solely on rendering the main content portion of the page without having to worry about the details of standard header, footer and navigation menu (commonly referred to as the 'c-frame').  For more information about the HPWeb 1.2 standard see the HP Web Standards site.

Everything outlined in red in the picture below will be rendered by the HPWeb layout.  Everything NOT outlined in red is referred to as the "page content".

The HPWeb layout provides a set of custom JSP tags to render the various HPWeb design elements as well as a SiteMesh-based servlet filter that can be used to automatically apply the layout to page.  Note that, because the layout is applied via a servlet filter, it can be used in conjunction with any application framework (Apache Struts, Spring MVC, JavaServer Faces, etc.) or view technology (JSP, Velocity, Freemarker, etc.).

The rest of this document explains how to configure the layout servlet filter and handle application-specific layout customizations.

2. System Requirements

The HPWeb layout requires a Java application server that support Java 2 Standard Edition (J2SE) version 1.4 and Java 2 Enterprise Edition (J2EE) version 1.4.  Note that implementation of the full J2EE specification is not required -- the specific pieces of J2EE 1.4 that are required are:

  • Java Servlet 2.4 Specification
  • JavaServer Pages 2.0 Specification

Application servers that meet the minimum requirements include Apache Tomcat 5.5.x and BEA WebLogic Server 9.0.

3. Installation

Use of the HPWeb layout requires the following JARs be placed in the web application's /WEB-INF/lib directory:

  • hpweb-core-x.x.x.jar
  • jstl-1.1.2.jar
  • oro-2.0.8.jar
  • sitemesh-2.3.jar
  • standard-1.1.2.jar
  • wpa-layout-manager-x.x.x.jar

In addition, the decorator templates named hpweb.jsp, hpweb-enhanced.jsp and hpweb-printable.jsp need to be placed in the /WEB-INF/decorators directory while the SiteMesh configuration file named decorators.xml needs to by placed in the /WEB-INF directory.

After all the necessary files have been added to the web application, the final installation step is to register the page filter in the deployment descriptor (web.xml file).  It is this page filter that is responsible for intercepting the outgoing responses and decorating them with the HPWeb look-n-feel.  The sample web.xml file shown below shows how the page filter should be registered:

<?xml version="1.0" encoding="ISO-8859-1"?>

<web-app
    xmlns="http://java.sun.com/xml/ns/j2ee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-app_2_4.xsd"
    version="2.4">

    <filter>
        <filter-name>sitemesh</filter-name>
        <filter-class>
            com.hp.frameworks.wpa.layout.filter.PageFilter
        </filter-class>
    </filter>

    <filter-mapping>
        <filter-name>sitemesh</filter-name>
        <url-pattern>/*</url-pattern>
        <dispatcher>REQUEST</dispatcher>
    </filter-mapping>

</web-app>

The only thing that may need to be customized in the example above is the <url-pattern> associated with the <filter-mapping>.  This URL pattern identifies the requests to which the filter should be applied.  In this example, the filter will be applied to ALL incoming requests (note the use of the * wildcard character).  Different URL patterns may be appropriate for different applications -- for a traditional Apache Struts application the URL pattern may be *.do.

4. Usage

Once the page filter is properly configured, the application developer simply creates pages just as they normally would with whatever application framework they are using.  The page filter will read the HTML stream that is created and apply the HPWeb layout automatically.

Below is a sample JSP page that might be created as part of a web application (note that the use of JSPs is not required, however it is a view technology that is commonly used with Java-based frameworks)

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

<%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>

<html>
    <head>
        <title>My Sample Page</title>
    </head>

    <body>
        <p>This is the content of my page</p>
    </body>
</html>

When the page filter receives this HTML it will parse out the content appearing between the <body> and </body> tags and insert into into the appropriate place within the HPWeb layout.  Likewise, the contents of the <head> block will be inserted into the <head> block of the HPWeb-decorated page that is ultimately returned to the user's browser.

NOTE: In order for the HPWeb look-n-feel to render properly it is important that you include the HTML 4.01 Transitional DOCTYPE at the top of all of your JSP pages.  Additionally, you must use the @page directive to indicate that your page content is encoded using the UTF-8 character set.  See the example above for the specific DOCTYPE and @page directive you must include in your pages.

5. Layout Customization

A good portion of the markup rendered by the HPWeb layout is static HTML, however, there are a number of design elements which are typically customized on a site-by-site or even page-by-page basis.  The two items were are most typically customized are the page title (displayed just to the right of the HP logo) and the left navigation menu.

For most of the configurable page elements, there are two different mechanisms available for specifying a custom value: 1) a declarative mechanism utilizing HTML <meta> or <content> tags, and 2) a programmatic mechanism utilizing the request-scoped {@link com.hp.frameworks.wpa.hpweb.HPWebModel} model object.  It is up to the application developer to choose which approach works best with the architecture of their application. 

5.1 Declarative Customization

The declarative approach typically involves the use of one or more <meta> tags within the <head> of the page that is being decorated.  The meta tags are parsed by the page filter and passed as arguments to the layout template so they can be used to drive the rendering behavior.  The following example shows how a page might specify a custom page title and feedback link.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

<%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>

<html>
    <head>
        <title>My Sample Page</title>

        <meta name="hpweb.title" content="My Custom Page Title">
        <meta name="hpweb.feedbackText" content="Leave feedback here">
        <meta name="hpweb.feedbackUrl" content="http://www.somesite.com"> 
    </head>

    <body>
        <p>This is the content of my page</p>
    </body>
</html>

The three meta tags in the example above are used to specify custom values for the pageTitle, feedbackText and feedbackUrl properties. Note that, when specified declaratively, each of the property names must be prefixed with the "hpweb." namespace in order to avoid collisions with any of layouts that may be installed. The complete list of valid meta tags will be described in the Configuration Reference section.  All meta tags will take the form of <meta name="property name" content="value"> where "property name" is one of the pre-defined property names and "value" is string value to be assigned to that property.

After being decorated by the HPWeb layout, the page shown in the example above would be rendered as follows:

Note that the page title and feedback link reflect the values of the meta tags specified in the original page.

Some of the configurable page elements require complex HTML strings that would be inconvenient to specify via simple <meta> tags.  For these situations, developers can use the <content> tag.  Note that the <content> tag is not a valid HTML tag, but will be parsed by the page filter and passed to the HPWeb layout.

The following example shows how a page might specify a custom locale selector using the <content> tag.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

<%@ page contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>

<html>
    <head>
        <title>My Sample Page</title>

        <meta name="hpweb.title" content="My Custom Page Title">
        <meta name="hpweb.feedbackText" content="Leave feedback here">
        <meta name="hpweb.feedbackUrl" content="http://www.somesite.com"> 
    </head>

    <body>

        <content tag="hpweb.localeSelector">
            <select name="locale">
                <option>US - English</option>
                <option>Canada - English</option>
            </select>
        </content>

        <p>This is the content of my page</p>
    </body>
</html>

Note how the application developer can use the body of the <content> tag to pass-in a complex HTML strings.  The tag attribute on the <content> tag identifies the layout property that the content should be associated with.  In the example above, an HTML <select> list is associated with the localeSelector property.

After being decorated by the HPWeb layout, the page shown in the example above would be rendered as follows (note the new locale selector being rendered at the top-right of the page):

Each page element that allows for declarative configuration can be specified using either a <meta> tag or a <content> tag. 

5.2 Programmatic Customization

As an alternative to the declarative approach, developers may also choose to make layout customizations programmatically by manipulating the {@link com.hp.frameworks.wpa.hpweb.HPWebModel} object.  For every incoming request that is processed by the page filter, a new instance of the HPWebModel object will be placed in a request-scoped variable.  The HPWebModel instance can be retrieved by application code at run-time and customized via the various properties that it exposes.  Those properties and their corresponding values will be made available to the layout template in much the same way as the meta tag values described above.

The following example shows how the HPWebModel might be accessed to perform the same customizations shown in the meta tag example above:

HPWebModel model = (HPWebModel) request.getAttribute("com.hp.frameworks.wpa.hpweb.model");
model.setTitle("My Custom Page Title");
model.setFeedbackText("Leave feedback here");
model.setFeedbackUrl("http://www.somesite.com");

5.3 HPWeb Enhanced Layout

As part of the HPWeb Hockney release in August 2007, the HPWeb team defined a new, "enhanced" version of the HPWeb design guidelines.  The enhanced version of the HPWeb layout relies on standards-based HTML and CSS for positioning instead of the HTML tables that are used for the the traditional layout.  Additionally, the enhanced layout provides a more contemporary look for some of the standard page elements like the left navigation menu, content units and tab sets.

When the enhanced styling is applied to the page shown at the beginning of this document, it would be rendered as follows:

With the exception of the left navigation menu and the content units, this page looks almost identical to the page rendered with the traditional layout; however, the use of standards-based HTML and CSS for positioning results in a page size that is less than half of the original.  Applications for which "page weight" and performance are a concern are encouraged to use the enhanced layout in lieu of the traditional layout.

There are a few different ways that the enhanced layout can be enabled.  To enable the enhanced layout for all pages in a site, the /WEB-INF/decorators.xml file can be altered to read as follows:

<?xml version="1.0" encoding="ISO-8859-1"?>

<decorators defaultdir="/decorators">

    <decorator name="hpweb" page="/WEB-INF/decorators/hpweb.jsp" />

    <decorator name="hpweb-enhanced" page="/WEB-INF/decorators/hpweb-enhanced.jsp">
        <pattern>/*</pattern>
    </decorator>

    <decorator name="printable" page="/WEB-INF/decorators/hpweb-printable.jsp" />

</decorators>

Note that a URL pattern of "/*" has been added to the hpweb-enhanced decorator definition, this instructs SiteMesh to apply the enhanced layout to all requests.

Alternatively, the enhanced layout can be enabled on a per-page basis by simply adding the following meta tag to the page:

<meta name="decorator" content="hpweb-enhanced" />

This instructs the SiteMesh page filter to apply to enhanced layout to just the page containing this meta tag.

Developers using the HPWeb tag library within the body of their page will need to include a reference to the enhanced tag library as follows:

<%@ taglib prefix="hpweb" uri="http://frameworks.hp.com/wpa/hpweb-content-enhanced" %>

More information about the HPWeb tag library can be found in the Custom Tags section below.

When using the enhanced layout, there are two different header styles to choose from: the jewel logo style (shown in the screenshot above) or the stretch logo style (shown in the screenshot below).

Furthermore, when using the stretch logo style, two different page widths can be selected: a "narrow" page width (740 pixels) or a "wide" page width (974 pixels). The narrow page style is shown above and the wide page style is shown below.

When using the stretch logo header style, the layout will default to the wide page style but developers can force the use of the narrow page style via the page width property.

To review, there are three variants of the enhanced layout: classic header, stretch logo header w/ narrow page, and stretch logo header w/ wide page. Not all of the configuration options described in the next section are applicable depending on the variant that is selected. For example, the site-specific search configuration is not available when using the stretch logo header, whereas the section title configuration is only available when using the stretch logo header.

6. Configuration Reference

The following sections describe all of the configurable page elements along with the corresponding meta/content tag and HPWebModel property names.

There are a number of layout variations to choose from and not all of the configuration properties listed below are applicable to each variation. Consult the "Applies To" field in each of the sections below to learn which layout styles a particular configuration item applies to. For reference, here are the layout variations that are available:

  • Classic Layout (use hpweb.jsp decorator)
  • Enhanced Layout (use hpweb-enhanced.jsp decorator)
    • Jewel Logo (default header style for enhanced layout)
    • Stretch Logo (enabled via header style configuration property)
      • Wide Page (default width when stretch logo header enabled)
      • Narrow Page (enabled via the page width configuration property)

6.1 Theme Color

The HPWeb theme color to be applied to the page. Valid values are listed below:

  • #990000 - Burgundy
  • #CC0066 - Magenta
  • #336666 - Sea Foam
  • #003366 - Navy
  • #336633 - Evergreen
  • #EB5F01 - Tangerine
  • #0066FF - Blue
  • #4FAF00 - Lime
  • #FFCC00 - Yellow
  • #FF0000 - Compaq Red
Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.theme"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setThemeColor(String)}

6.2 Header Style

By default, the HPWeb header contains four different regions: the top bar (containing the HP Passport links and the country/language indicator), the global navigation menu, the search bar (which also contains the "Contact HP" link) and the logo/title area.  As part of the enhanced design standards introduced as part of the HPWeb Hockney release, the HP.com team defined an alternate header style which removes the global navigation menu and uses a new "stretch" design treatment for the logo/title area.

Developers wishing to use the "stretch" header as part of their application need to first ensure that they are using the HPWeb Enhanced layout (see the HPWeb Enhanced Layout section above) and then specify a value of "stretch" for the header style property.  At this point, the only valid value for the header style property is "stretch"; any other value will result in the standard header being rendered.

Applies To: Enhanced Layout
Tag Name: "hpweb.headerStyle"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setHeaderStyle(String)}

6.3 Page Width

When using the enhanced layout in conjunction with the stretch logo header, the page width property is used to indicate whether the narrow (740 pixels) or wide (974 pixels) page style should be used. The default value of this property is "wide". A value of "narrow" should be specified to force the rendering of the narrow page width. Other than the width of the rendering area, the primary difference between these two options is that the sets of links ("Live Chat", "Other ways to buy", "Order Status", etc.) which are rendered in the logo area of the wide page width are not available in the narrow page width.

Applies To: Enhanced Layout w/ Stretch Logo Header
Tag Name: "hpweb.pageWidth"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setPageWidth(String)}

6.4 Username

If specified, the username will be rendered as part of the welcome message at the top left of the page.  A non-empty value for the username property indicates that the user is logged-in and will determine whether the sign-in/register links are displayed or whether the sign-out/profile links are displayed.

Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.username"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setUsername(String)}

6.5 HP Passport Sign-in URL

The fully-qualified URL to the HP Passport sign-in page.  This URL will be used in conjunction with the "Sign-in" link rendered at the top of the page.  The value for this property is also used to indicate whether or not the page should render any of the HP Passport-related links: a non-null value indicates that the page is integrated with HP Passport while a null value indicates that none of the HP Passport-related links should be rendered.

Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.signInUrl"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setSignInUrl(String)}

6.6 HP Passport Sign-out URL

The fully-qualified URL to the HP Passport sign-out page.  This URL will be used in conjunction with the "Sign-out" hyperlink rendered at the top of the page.  The "Sign-out" hyperlink is only rendered when the "signInUrl" and "username" properties both contain non-null values.

Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.signOutUrl"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setSignOutUrl(String)}

6.7 HP Passport Register URL

The fully-qualified URL to the HP Passport registration page.  This URL will be used in conjunction with the "Register" hyperlink rendered at the top of the page.  The "Register" hyperlink is only rendered when the "signInUrl" property contains a non-null value and "username" property is empty (indicating that the user is not logged-in).

Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.registerUrl"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setReigsterUrl(String)}

6.8 HP Passport Edit Profile URL

The fully-qualified URL to the HP Passport profile page.  This URL will be used in conjunction with the "Edit your profile" hyperlink rendered at the top of the page.  The "Edit your profile" hyperlink is only rendered when the "signInUrl" and "username" properties both contain non-null values.

Applies To: Classic and Enhanced Layouts
Tag Name: "hpweb.profileUrl"
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setProfileUrl(String)}

6.9 My Account Profile URL

When using the enhanced layout with the stretch logo header developers have the option of replacing the HP Passport "Register" and "My Profile" links with a link that reads "My Account". This property is used to specify the destination URL for the "My Account" link.  If a value is specified for this property, the "My Account" link will be rendered in place of the HP Passport "Register" and "My Profile" links. 7. Left Menu XML

Many applications require that the same left navigation menu be rendered for all pages in the site.  For these situations it may be inefficient to define the left navigation menu using either the declarative or programmatic approaches described above.  As an alternative, developer's can define their left navigation menu in a simple XML file and have it applied to all pages in their site.

A left navigation menu can be defined by creating a file named hpweb-leftmenu.xml and placing it in the default package of the application's classpath (for most web applications this means placing the XML file in the /WEB-INF/classes directory).  Below is a sample XML file that defines the same left navigation menu shown in the examples above:

<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE menu PUBLIC
    "WPA_HPWEB_LEFTMENU_1_0_DTD"
    "http://frameworks.hp.com/wpa/hpweb/left-menu_1_0.dtd">

<menu>
    <menuitem id="menu1" title="Heading 1">
        <menuitem id="menu1.1" title="Link 1" url="http://www.hp.com" />
   </menuitem>
    <menuitem id="sep1" />
    <menuitem id="menu2" title="Link 2" url="http://www.hp.com" />
</menu>

A few things to note about the example above:

  • The XML DOCTYPE must be specified exactly as shown above
  • All <menuitem> elements must be nested within a top-level <menu> element
  • All <menuitem> elements must specify a unique value for the id attribute
  • Any <menuitem> elements with an empty title attribute will be rendered as menu separators
  • Nested sub-menus are declared by nesting <menuitem> elements within other <menuitem> elements.

At run-time the left menu XML file will be used to populate the leftMenuItems property of the HPWebModel object.  This means that developers can make run-time modifications to the left menu by accessing the HPWebModel object -- this is useful in situations where certain menu items may need to be hidden/shown depending on the current context of the user.

The complete DTD for the left menu XML file can be seen below:

<!--
    A "Boolean" is the string representation of a boolean (true or false)
    variable.
-->
<!ENTITY % Boolean "(true|false)">

<!--
    The "menu" element is the root of the configuration file
    hierarchy. No attributes are defined.
-->
<!ELEMENT menu (menuitem)+>


<!--
    The "menuitem" element indicates that a menu item should be inserted into
    the left navigation menu. A "menuitem" can have any number of nested
    "menuitem"s.

    The following attributes are defined:

    id       Unique identifier for this menu item.

    title    String to use as hyperlink text.

    url      URL to use for hyperlink.

    highlighted   Boolean indicating whether or not this menu item should be
             rendered in the active (or highlighted) state.

    visible  Boolean indicating whether or not this menu item is visible.
-->
<!ELEMENT menuitem (menuitem*)>
<!ATTLIST menuitem id ID #REQUIRED>
<!ATTLIST menuitem title CDATA #IMPLIED>
<!ATTLIST menuitem url CDATA #IMPLIED>
<!ATTLIST menuitem highlighted %Boolean; #IMPLIED>
<!ATTLIST menuitem visible %Boolean; #IMPLIED>

8. Printable Version

The HPWeb Layout is also capable of rendering pages that conform to the HP.com "printable version" standards.  The printable version layout is used to make pages suitable for printing on standard Letter- or A4-sized paper.

The HPWeb printable version rules state that the following changes should be made to the page:

  • Remove all top navigation including the global navigation buttons, the contact/search bar and all links/information above the global navigation buttons including the HP Passport links and language/country indicator.
  • Remove the left navigation menu including promotions
  • Remove all background colors (except for gray background in table rows and content units)
  • Remove the printable version links
  • Deactivate all links except those in the page footer
  • Change all blue hyperlink text to black
  • Disable form submits
  • Remove tab corners and background colors that create navigation tabs
  • Change white header text to black
  • Remove all theme colors
  • Shift the bottom banner to the left
  • Insert "Return to original page" links that will take users back to the non-printable version of the current page

9.1 Content Unit Tag

The content unit tag is used to highlight blocks of information on the page.  For complete information about how and when the content unit design element should be applied, please see the HP Web Standards.

The contentUnit tag supports the following attributes:

nametypedescription
title java.lang.String Title text to display within the content unit header.  This is a required value.
url java.lang.String The fully-qualified URL for the title text hyperlink.
width java.lang.Integer Width in pixels of the content unit.  A value of 0 is interpreted to mean "100%".  The default value is 0.
headerLevel java.lang.Integer Indicates which header markup should be applied to the content unit title text.  For example, a value of 2 means that the title text will be marked up using the <h2> tag.  Valid values are 2 through 6.  The default value is 2.
style java.lang.String Indicates the style of content unit to be rendered.  When using the standard HPWeb tag library, valid values for this attribute are:
  • intro
  • levelA
  • levelABig
  • levelB
  • levelC
  • levelCShaded
  • levelD

When using the HPWeb Enhanced tag library, valid values for this attribute are:

  • intro
  • levelA
  • levelB
  • levelBBeveled
  • levelC
  • levelD
  • levelE
The default value is "intro". See below for examples of the various styles.

The code to render a 400 pixel Level A content unit would be as follows:

<hpweb:contentUnit title="My Title String" width="400" style="levelA">

9.2 Tab Set Tag

A tab set is a convenient mechanism for presenting multiple views of data related to a particular concept.  For complete information about how and when the tab set design element should be applied, please see the HP Web Standards.

Rendering a tab set actually requires the use of two different tags: the tab tag and the tabSet tag.

<hpweb:tabSet width="400" selectedIndex="2">
    <hpweb:tab title="Tab 1" url="mypage?tab=0">tab 1 content area</hpweb:tab>
    <hpweb:tab title="Tab 2" url="mypage?tab=1">tab 2 content area</hpweb:tab>
    <hpweb:tab title="Tab 3" url="mypage?tab=2">tab 3 content area</hpweb:tab>
    <hpweb:tab title="Tab 4" url="mypage?tab=3">tab 4 content area</hpweb:tab>
</hpweb:tabSet>

The tabSet tag defines the group of tabs and then individual tab tags are nested within the tab set to define the content.  The tab set defined in the code above would be rendered as follows with the standard HPWeb tag library.

The tabSet tag supports the following attributes:

nametypedescription
width java.lang.Integer Width in pixels of the tab set.  A value of 0 is interpreted to mean "100%".  The default value is 0.
selectedIndex java.lang.Integer The zero-based index of the tab that should be rendered as the active tab.  The default value is 0.
stretchTabs java.lang.Boolean Flag indicating whether or not the individual tabs should be stretched so that they occupy the full width of the tab set.  The default value is "true".  Note that this attribute has no impact when the enhanced tag library is used.
hairline java.lang.Boolean Flag indicating whether or not the hairline border should be rendered around the right and bottom sides of the tab content area.  The default value is "true".

The tab tag supports the following attributes:

nametypedescription
title java.lang.String Title text to display for the tab.  This is a required value.
url java.lang.String The URL for the title text hyperlink.
alt java.lang.String Short description or advisory text to associate with the tab's title text.

Any content present between the open and close <hpweb:tab> tags becomes the content of the rendered tab.  Note that only the selected tab (whose index is specified with the selectedIndex attribute on the tabSet tag) will have its content rendered.

9.3 Right Menu Tag

The right navigation menu is an optional page element that can be used to display links that are related to the main page content.  For complete information about how and when the right navigation menu design element should be applied, please see the HP Web Standards.

Rendering a right navigation menu requires the use of two different tags: the rightMenu tag and the rightMenuItem tag.

<hpweb:rightMenu>
    <hpweb:rightMenuItem title="Heading 1">
        <hpweb:rightMenuItem title="Link 1" url="http://www.hp.com" />
        <hpweb:rightMenuItem title="Link 2" url="http://www.hp.com" />
    </hpweb:rightMenuItem>
    <hpweb:rightMenuItem />
    <hpweb:rightMenuItem title="Link 3" url="http://www.hp.com">
        <hpweb:rightMenuItem title="Link 4" url="http://www.hp.com" />
        <hpweb:rightMenuItem title="Link 5" url="http://www.hp.com" />
    </hpweb:rightMenuItem>
</hpweb:rightMenu>

The rightMenu tag defines the overall menu while the rightMenuItem tags represent the individual elements of the menu.  In the example above, seven menu items have been specified as follows:

  • A non-clickable heading displaying the text "Heading 1"
  • Two clickable menu items nested under the "Heading 1" item
  • A menu separator (separators are specified by leaving the title attribute empty)
  • A clickable heading displaying the text "Link 3"
  • Two clickable menu items nested under the "Link 3" item

nametypedescription
promotion  java.lang.String Block of HTML that should be rendered in the promotion area at the bottom of the right navigation menu.  If no value is specified, no promotion content will be rendred.

The rigthMenuItem tag supports the following attributes:

nametypedescription
title  java.lang.String Title text to display for the menu item.  If no value is specified for this attribute, a menu separator will be rendered.  Note that menu separators can only appear as top-level menu items (i.e. separators as nested menu items are not allowed).
url java.lang.String The URL for the hyperlink.  If no value is specified for this attribute, the menu item will be rendered as a non-clickable heading.  Note that non-clickable headings can only appear as top-level menu items (i.e. headings as nested menu items are not allowed).

 If you have a large block of HTML that you'd like to specify for the promotion attribute on the rightMenu tag, it may be more convenient to use the JSP attribute tag as follows:

<hpweb:rightMenu>
    <jsp:attribute name="promotion">
        <strong>HELLO WROLD</strong>
    </jsp:attribute>
    <jsp:body>
        <hpweb:rightMenuItem title="Heading 1">
            <hpweb:rightMenuItem title="Link 1" url="http://www.hp2.com" />
            <hpweb:rightMenuItem title="Link 2" url="http://www.hp2.com" />
        </hpweb:rightMenuItem>
        <hpweb:rightMenuItem />

            <hpweb:rightMenuItem title="Link 5" url="http://www.hp2.com" />
        </hpweb:rightMenuItem>
    </jsp:body>
</hpweb:rightMenu>

Note that, when you specify the value for the promotion attribute in this way you must wrap the rightMenuItem tags in a JSP body tag as shown above.

    </hpweb:rightMenu>
