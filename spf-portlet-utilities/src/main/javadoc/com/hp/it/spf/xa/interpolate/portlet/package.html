This package contains plain Java classes for interpolating dynamic tokens in text. The interpolation tokens supported in this package are:

{AFTER:date}
Includes the surrounded content after the particular date.
{BEFORE:date}
Includes the surrounded content until the particular date.
{CONTENT-URL:path}
Replaced with the properly-encoded URL for the external or internal static resource file named path.
{COUNTRY-CODE}
Replaced with the ISO 3166-1 country code for the current portlet request.
{COUNTRY-CODE:case}
As above, forced to lower or upper case (default is upper).
{EMAIL}
Replaced with the email address for the user in the current portlet request.
{GROUP:groups}
Includes the surrounded content if the user in the current portlet request is a member of one of the named groups. Otherwise suppresses the surrounded content.
{HPP-LANGUAGE-CODE}
Replaced with the HP Passport language code for the current portlet request.
{HPP-LANGUAGE-CODE:case}
As above, forced to lower or upper case (default is lower).
{INCLUDE:key}
Replaced with the value for the key property as found in the token substitutions file (eg default_includes.properties by default).
{LANGUAGE-CODE}
Replaced with the ISO 639-1 language code for the current portlet request.
{LANGUAGE-CODE:case}
As above, forced to lower or upper case (default is lower).
{LANGUAGE-TAG}
Replaced with the RFC 3066 language tag for the current portlet request.
{LANGUAGE-TAG:case}
As above, forced to all lower or all upper case (default is mixed case: lowercase language and uppercase country).
{LOCALIZED-CONTENT-URL:path}
Replaced with the properly-encoded URL for the (possibly localized) external or internal static resource file, the base name of which is path.
{LOGGED-IN}
Includes the surrounded content if the user in the current portlet request is logged-in (ie, not anonymous). Otherwise suppresses the surrounded content.
{LOGGED-OUT}
Includes the surrounded content if the user in the current portlet request is not logged-in (ie, is anonymous). Otherwise suppresses the surrounded content.
{NAME}
Replaced with the full name for the user in the current portlet request, displayed in customary order for the locale.
{PORTLET:portlets}
Includes the surrounded content if the current portlet is one of the named portlets. Otherwise suppresses the surrounded content.
{REQUEST-URL}
Replaced with the complete URL for the current portal request which targeted this portlet.
{REQUEST-URL:spec}
Replaced with the complete URL for the current portal request which targeted this portlet, with scheme and port adjusted as specified.
{ROLE:roles}
Includes the surrounded content if the user in the current portlet request is in one of the named roles. Otherwise suppresses the surrounded content.
{SITE}
Replaced with the name of the current portal site which includes this portlet.
{SITES:names}...{/SITES}
Includes the surrounded content if the current request is for one of the portal site names. Otherwise suppresses the surrounded content.
{SITE-URL}
Replaced with the home-page URL for this portlet's portal site.
{SITE-URL:spec}
Replaced with the page URL for the given friendly URI at this portlet's portal site. Scheme and port can also be specified.
{URL-ENCODE:string}
Replaced with the URL-encoded version of the given string.
{USER-PROPERTY:key}
Replaced with the named user property key in the current portlet request.

FileInterpolator

The {@link com.hp.it.spf.xa.interpolate.portlet.FileInterpolator} class lets you read and interpolate the content of a text file. The file may be stored as either an external or internal resource file. (External files are in the portlet resource bundle directory while internal files are inside your portlet WAR. See the SPF Portlet Utilities Developer's Guide for more discussion about external versus internal resources.) The file may be localized; the class looks for the best-fit localized version of the file when loading it. Please see the class documentation for more information.

TokenParser

The {@link com.hp.it.spf.xa.interpolate.portlet.TokenParser} provides parsing methods for strings, for each of the above listed token types. This class is used by the {@link com.hp.it.spf.xa.interpolate.portlet.FileInterpolator} and if you just use the FileInterpolator you will never need to use the TokenParser directly. Please see the class documentation for more information.
