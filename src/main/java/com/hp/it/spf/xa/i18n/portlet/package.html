This package contains Java utilities for internationalizing Shared Portal Framework portlet applications.

I18nUtilities

The {@link com.hp.it.spf.xa.i18n.portlet.I18nUtilities} class provides methods for:

  • accessing message resources using Spring (augmented with inherent SPF capabilities such as the <contextual_help>...</contextual_help> and <no_localization>...</no_localization> tokens)
  • accessing other static resource files from the portlet resource bundle folder (see below) or your portlet WAR
  • and more.
Please see the class documentation for more information.

Where you put your resources

You may put your message and other static resource files (ie message properties and images, HTML files, video files, PDF's, etc) into your portlet WAR where they traditionally go:

  • Your message properties would go into your WEB-INF/classes/, optionally under some additional sub-path (we recommend portlet/i18n/ for reasons that will become clear). Be sure to configure Spring properly so it can find them. For example, if you use Spring's {@link org.springframework.context.support.ReloadableResourceBundleMessageSource} class, you would put this into your application context configuration (eg either your application-wide applicationContext.xml or your portlet-specific version of that file) for a bundle of message properties with basename of messages (eg messages.properties, messages_fr_FR.properties, etc):

    	<bean id="messageSource"
		class="org.springframework.context.support.ReloadableResourceBundleMessageSource">
		<property name="basenames">
			<list>
				<value>
					classpath:portlet/i18n/messages
				</value>
			</list>
		</property>
	</bean>

  • Your other static resource files (images, videos, PDF's, etc) would go into respective folders at the root of your portlet WAR. For example, if you have a bundle of HTML files (text.html, text_es_MX.html, etc) and a bundle of images (picture.jpg, picture_ja.jpg, etc), they could go into your portlet WAR as follows:

    	<root>
	    html/
	        text.html
	        text_es_MX.html
	        ...
	    images/
	        picture.jpg
	        picture_ja.jpg
	        ...

Then when you use the standard Spring message API's, or the SPF portlet message API's in I18nUtilities, or the tag libraries for either, they will find your message resources inside your portlet WAR. And when you use the localized file access API's in I18nUtilities, they will find your statis resource files there likewise.

However, you may instead put your messages and other resources outside of your portlet WAR. Doing so may have some advantages - for example, it may make for easier administration of the resources. Currently the SPF supports you putting them into what is called the portlet resource bundle folder, which is any folder on the portlet server's filesystem to which the portlet JVM has at least read access. By default, /opt/sasuapps/spf/globalResources/portlet/i18n is assumed as the path for the portlet bundle folder, but you can override that by configuring i18n_portlet_config.properties (see below).
