HP.com specific locale negotiation components.

{@link com.hp.fast.web.hpweb.LocaleFilter}, a servlet filter, is the primary class of interest to applications. It implements all HP.com requirements regarding user locale resolution. The filter intercepts incoming requests, gathers all locale information available in the request, compares it against application locale availability and selects the best match.

Solutions for Spring MVC based applications are provided in the {@link com.hp.fast.web.spring} package.

Out-of-the-box the {@link com.hp.fast.web.hpweb.LocaleFilter} addresses all HP.com locale requirements. It is parameterized to enable application-specific HP Passport integration.

A web application will typically:

  1. Provide an implementation of {@link com.hp.fast.web.hpweb.LocaleProvider} that returns the locales in which the application is available.
  2. If the application is integrated with HP Passport:
    1. Provide an implementation of {@link com.hp.fast.web.hpweb.LocaleProvider} that returns the locale specified in the user's HP Passport profile. The way in which an application retrieves this information is specific to the HP Passport integration.
    2. Provide an implementation of {@link com.hp.fast.web.hpweb.LocaleProviderFactory} that returns instances of the locale provider defined in the previous step.
  3. Configure the {@link com.hp.fast.web.hpweb.LocaleFilter} in the application's web.xml including initialization parameters specifying the above implementation classes.

The following illustrates the configuration of the {@link com.hp.fast.web.hpweb.LocaleFilter}.

<filter>
	<filter-name>Locale Filter</filter-name>
	<filter-class>com.hp.fast.web.hpweb.LocaleFilter</filter-class>
	<init-param>
		<param-name>targetLocaleProvider</param-name>
		<param-value>com.hp.myapplication.MyApplicationLocaleProvider</param-value>
	</init-param>
	<init-param>
		<param-name>passportLocaleProviderFactory</param-name>
		<param-value>com.hp.myapplication.MyPassportLocaleProviderFactory</param-value>
	</init-param>
</filter>
