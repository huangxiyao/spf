1. Overview

Portal applications running on the Vignette Application Portal (VAP) product that require the look-and-feel of the HPWeb UI standards can make use of the HPWeb VAP components.  Using the HPWeb VAP Components allow developers to focus solely on rendering the main content portion of the page without having to worry about the details of a standard header, footer and navigation menu (commonly referred to as the 'c-frame').  For more information about the HPWeb 1.2 standard see the HP Web Standards site.

Everything outlined in red in the picture below will be rendered by the HPWeb layout.  Everything NOT outlined in red is referred to as the "page content".

The HPWeb layout is supported by the following components to render the layout and various HPWeb design elements in the layout:

  • Theme component
    • HPWeb Layout Theme
  • Grid components
    • HPWeb Layout System Grid
    • HPWeb Layout Horizontal Navigation Grid
    • HPWeb Layout Vertical Navigation Grid
    • HPWeb Layout Split Navigation Grid
  • Style type component
    • HP Layout Config Style Type
  • Style components
    • HPWeb Layout Account Controls Style
    • HPWeb Layout Header Style
    • HPWeb Layout Footer Style
    • HPWeb Layout Horizontal Navigation Style
    • HPWeb Layout Vertical Navigation Style
    • HPWeb Layout Portlet Chrome Styles
      • Introduction Box Style
      • Level Alpha Header Style
      • Level Beta Header Style
      • Level Beta Header with Bevel Style
      • Level Gamma Header Style
      • Chromeless (Callout) Style
    • HPWeb Layout Config Style
See the HPWeb Components section for more information.

2. System Requirements

The HPWeb components runs on Vignette Application Portal (VAP) product version 7.4 and above.  Java Servlet 2.4, JSP 2.0 and JSTL 1.1 are required.

The following JARs are required and expected to be in the portal application's /WEB-INF/lib directory:

  • jstl.jar (version 1.1.x)
  • oro.jar (version 2.0.x)
  • standard.jar (version 1.1.x)
Since the VAP product provides these JARs, the HPWeb Components CAR file will not include them.

3. Installation

The HPWeb components are packaged in a HPWebComponents-<version>.car file which is downloadable from the WPA HPWeb VAP Components Downloads directory.   Download the latest CAR file version to a local temporary directory.  

Use the VAP Server Console to install/import the components.   After they are imported, share each HPWeb component to the site(s) where they will be used; otherwise, they won't be accessible. Since the HPWeb Components contain class JAR files the application server has to be restarted twice to load the classes.

Note: Installation of the HPWeb libraries will placed the following JARs into the portal web application's /WEB-INF/lib directory:

  • hpweb-core.jar
  • hpweb-resources.jar
  • hpweb-vap.jar
  • wpa-layout-manager.jar

3.1 Vignette Page Display Issue

There is a known Vignette issue under certain conditions using the default configured DHTML Page Display secondary page and it affects the HPWeb horizontal navigation and portlet horizontal positioning on IE6 and Firefox browsers.

If you do not plan to implement your own Page Display secondary page, then perform the following steps to use the Page Display secondary page instead of the DHTML Page Display secondary page:

  1. Go to Server Console -> Components -> Pages -> Secondary Page List . On the page, click on the "Page Display" secondary page checkbox, select "Share to..." from the combo box and then the "Do It" button. Then select the site(s) that you want to share to.
  2. Go to the Site Console for that site -> Site Settings -> Appearance -> Secondary Pages ->. For the "Page Display" Page Type ensure you select "Page Display" in the Use Secondary Page column.

4. Usage

Use the VAP Site Console to apply the HPWeb layout to a site or a set of pages by setting its theme to HPWeb Theme and its grid to one of the HPWeb grids. You can customize the HPWeb layout to your portal and pages through the component properties or programmatically. Component properties customization involves setting layout properties in the HPWeb Components while programmatic customization involves setting layout properties to the HPWebModel named request-scoped bean. See the Layout Customization section for more information.

5. HPWeb Components

5.1 Style Types

All of the page elements in the HPWeb c-frame match with the style types provided by VAP, except for the HP Layout Config style type. The HPWeb components include implementations for each of the following VAP style types:

  • Account Controls
  • Header
  • Footer
  • Horizontal Navigation
  • Vertical Navigation
  • Portlet Chrome
  • HP Layout Config

The HP Layout Config Style Type is provided for a style instance, e.g. HPWeb Layout Config Style, to hold custom code for programmatic layout configuration.

5.2 Themes

HPWeb components include the HPWeb Theme that aggregates the following styles:

  • HPWeb Account Controls Style
  • HPWeb Header Style
  • HPWeb Footer Style
  • HPWeb Vertical Navigation Style
  • HPWeb Horizontal Navigation Style
  • HPWeb Layout Config Style

5.3 Grids

Four HPWeb grids are provided:

  • HPWeb System Grid - This grid is used for system pages and pages that do not require any navigation. Neither the vertical navigation nor the horizontal navigation is displayed on this grid.
  • HPWeb Horizontal Navigation Grid - This grid is used for pages that require a horizontal navigation menu but no vertical navigation menu.
  • HPWeb Vertical Navigation Grid - This grid is used for pages that require a vertical navigation menu but no horizontal navigation menu.
  • HPWeb Split Navigation Grid - This grid is used for pages that require both horizontal and vertical navigation menus. Depending on the navigation split level that is set in the VAP Site Console, the navigation items above the navigation split level are displayed in the horizontal menu while the navigation items below the navigation split level are displayed in the vertical navigation menu.

5.4 Styles

5.4.1 HPWeb Account Controls Style

This top-most level style on the page displays:

  • Welcome message
  • Navigation controls that are common throughout the site including sign-in/sign-out link and register/edit profile link
  • User locale text or placeholder for locale selector html
The Page Width configuration property affects whether the navigation controls are displayed in the accounts controls area or in the header area.

5.4.2 HPWeb Header Style

The HPWeb Header Style implements the stretch HP logo style that displays:

  • Stetch HP logo with section title, contact HP link, sign-in/sign-out link, register/edit profile link, live chat link, shopping cart links, etc
  • HP-company wide links and search area
  • Page title, title tagline, breadcrumbs, and top promotion area

5.4.3 HPWeb Footer Style

The HPWeb Footer Style displays:

  • Privacy link, terms and conditions link, feedback link, site map link, etc

5.4.4 HPWeb Horizontal Navigation Style

The HPWeb Horizontal Navigation Style displays a three-level horizontal navigation as configured in the VAP Site Console for the site. It is placed below the HP-company wide links area and above the page title of the header.

It is composed of:

  • Primary navigation display as tabs
  • Secondary navigation display as buttons for the selected (parent) tab
  • Third-level navigation display as a flyout menu of items under the hovered (parent) button
The site help link is displayed in the rightmost portion of the secondary navigation and is enabled by setting the Help Text and Help Url properties.

5.4.5 HPWeb Vertical Navigation Style

The HPWeb Vertical Navigation Style displays:

  • Left menu title/link
  • Navigation left menu as configured in VAP Site Console, or created programmatically during run-time
  • Left promotion area

5.4.6 HPWeb Layout Config Style

The HPWeb Layout Config Style is the style where to include code for customizing the HPWeb layout. It must not contain HTML that outputs HTTP responses. Each HPWeb grid includes this style at the beginning of the grid's execution, so that the layout properties may be initialized before the HPWeb layout accesses them for rendering.

The custom code goes into the primary JSP file, hpweb_layout_config.jsp. Also included is the sample file, sample_hpweb_layout_config.jsp, which contains many sample customizations, including HPP Central Forms integration. See the next section for more information on layout customization.

Additionally, customized code for including in the <head> element can be placed in the hpweb_layout_config_head.jsp file. Code such as javascript code and CSS styles can be placed in this file. A sample file, sample_hpweb_layout_config_head.jsp is included.

5.4.7 HPWeb Portlet Chrome Styles

The HPWeb portlet chromes are used to display portlets in different hierarchical levels on a page. The different portlet header levels are the same as the enhanced hierarchy header elements described in the HP Web Standards site.

The following portlet chromes are provided:
  • Introduction Box
  • Level Alpha Header
  • Level Beta Header
  • Level Beta Header with Bevel
  • Level Gamma Header
  • Chromeless (Callout)

The portlet title string displayed in the chrome header is retrieved from the title property of the portlet in VAP, and if it does not have a value, the portlet name in VAP is used. The Chromeless portlet chrome does not display a portlet title. The header markup, e.g. <h2>, for the portlet title string is configurable with the Portlet Chrome Header Level property.

Level Alpha Header, Level Beta Header, and Level Beta Header with Bevel portlet chromes can be configured to have a gray outline around the portlet content or bounding box. To enable a bounding box, set the Portlet Chrome Bounding Box property to true.

If a portlet returns an exception, the default logic will display an error message and optionally an error code, if it exists, in the portlet content. The error message is retrieved from the properties using the return string from the portletRenderException.getName() method call as the key, and the error code is the return string from the portletRenderException.getErrorParameters() method call. The properties are retrieved from the secondary page that the portlet chrome is contained in. This is the default error display that is implemented in the portlet_render_error_inc.jsp file, and it can be replaced with custom error display code.

6. Layout Customization

A good portion of the markup rendered by the HPWeb layout is static HTML, however, there are a number of design elements which are typically customized on a site-by-site or even page-by-page basis.  One item that is most typically customized is the page title.

For most of the configurable page elements, there are two different mechanisms available for specifying a custom value: 1) a mechanism utilizing HPWeb component properties, and 2) a programmatic mechanism utilizing the HPWebModel request-scoped attribute object.  A property with a non-null value in the HPWebModel object will take precedence over a value in the component property. It is up to the application developer to choose which approach is preferred. The general rule is to use the component properties for static constant values and the programmatic mechanism for dynamic run-time values. It is not necessary to set values for all of the properties. Some properties can be enabled to auto-generate values at run-time, e.g. Left Menu and Breadcrumb Items.

6.1 Customization in Component Properties

This approach involves setting property values in the HPWeb components. See Configuration Reference for the list of properties and components.

6.2 Programmatic Customization

The com.hp.frameworks.wpa.portal.hpweb.HPWebModel bean type contains properties which are customizations to the HPWeb layout. The programmatic approach involves setting the property values on the HPWebModel named request-scoped attribute bean of that type. Each HPWeb grid provides an integration point by including the HPWeb Layout Config Style. The hpweb_layout_config.jsp primary file in the HPWeb Layout Config Style is where a portal can place its layout customization code.

The following sample code references the HPWebModel bean, and set HPWeb properties on it.

<jsp:useBean id="HPWebModel" scope="request" class="com.hp.frameworks.wpa.portal.hpweb.HPWebModel" />

<jsp:setProperty name="HPWebModel" property="themeColor" value="#0066FF" />
<jsp:setProperty name="HPWebModel" property="topPromotion" value="This is a top promotion" />

6.3 HPWeb Enhanced Layout

Starting with the HPWeb Hockney release in August 2007, the HPWeb team defined a new, "enhanced" version of the HPWeb design guidelines.  The enhanced version of the HPWeb layout relies on standards-based HTML and CSS for positioning instead of the HTML tables that are used for the the traditional layout.  Additionally, the enhanced layout provides a more contemporary look for some of the standard page elements like the left navigation menu, content units and tab sets.

The HPWeb Components supports to the latest enhanced layout standards with the HPWeb JasperJohns release (September 2008); however, it will not include earlier HPWeb standards like the "classic" layout of the HPWeb design guidelines and the "jewel logo" (versus the supported "stretch logo) header style.

When using the enhanced layout, there is one supported header styles: the stretch logo style (shown in the screenshot below).

Furthermore, when using the stretch logo style, two different page widths can be selected: a "narrow" page width (740 pixels) or a "wide" page width (974 pixels). The narrow page style is shown above and the wide page style is shown below.

When using the stretch logo header style, the layout will default to the wide page style but developers can force the use of the narrow page style via the Page Width property.

7. Configuration Reference

The following sections describe all of the configurable page elements along with their corresponding property names in the component properties and HPWebModel property setter methods. Note that the HPWebModel object is a JavaBean object. When you want to access the bean property in JSP using the JSP <jsp:getProperty> tag or expression language, the bean property name is xyzProperty when the setter method name is setXyzProperty().

There are a number of layout variations to choose from and not all of the configuration properties listed below are applicable to each variation. Consult the "Applies To" field in each of the sections below to learn which layout styles a particular configuration item applies to. For reference, here are the layout variations that are available:

  • Enhanced Layout (HPWeb Hockney release and later)
    • Stretch Logo (enabled via Header Style configuration property)
      • Wide Page (default width when stretch logo header enabled)
      • Narrow Page (enabled via the Page Width configuration property)

7.2 Header Style

The enhanced design standards define a header style with the top bar (containing the HP Passport links and the country/language indicator), "stretch" design treatment for the logo/title area (which also contains the "Contact HP" link), and global navigation and search bar.

At this point, the only valid value and default value for the header style property is "stretch".

Applies To: Enhanced Layout
Message Key (Component): "hpweb.headerStyle" (HPWeb Grid)
HPWebModel Property: {@link com.hp.frameworks.wpa.hpweb.HPWebModel#setHeaderStyle(String)}

7.3 Page Width

When using the enhanced layout in conjunction with the stretch logo header, the page width property is used to indicate whether the narrow (740 pixels) or wide (974 pixels) page style should be used. The default value of this property is "wide". A value of "narrow" should be specified to force the rendering of the narrow page width. Other than the width of the rendering area, the primary difference between these two options is that the sets of links ("Live Chat", "Other ways to buy", "Order Status", etc.) which are rendered in the logo area of the wide page width are not available in the narrow page width.

Applies To: Enhanced Layout
Message Key (Component): "hpweb.pageWidth" (HPWeb Grid)
HPWebModel Property: {@link com.hp.frameworks.wpa.portal.hpweb.HPWebModel#setPageWidth(String)}

7.4 Username

If specified, the username will be rendered as part of the welcome message at the top left of the page.  A non-empty value for the username property indicates that the user is logged-in and will determine whether the sign-in/register links are displayed or whether the sign-out/profile links are displayed.

Applies To: Enhanced Layout
Message Key (Component): none
HPWebModel Property: {@link com.hp.frameworks.wpa.portal.hpweb.HPWebModel#setUsername(String)}

7.5 HP Passport Sign-in URL

7.38 Breadcrumb Items

Breadcrumbs are used to represent the current page's location in the site hierarchy and are displayed immediately above the page header.  Breadcrumbs are optional page element. 

Breadcrumbs can be automatically generated by setting the Generate Breadcrumbs Items property to true on the HPWebModel object. The HPWeb layout will generate the breadcrumbs for the navigation items leading to the current page.

Alternatively breadcrumbs can be specified programmatically using the breadcrumbItems property on the HPWebModel object. The breadcrumbItems property accepts a list of {@link com.hp.frameworks.wpa.hpweb.MenuItem} objects.  Note that only objects of type com.hp.frameworks.wpa.hpweb.MenuItem should be added to the list.

The following example shows how to programmatically add two breadcrumbs to the HPWebModel.

<jsp:useBean id="HPWebModel" scope="request"
    class="com.hp.frameworks.wpa.portal.hpweb.HPWebModel" />

<jsp:scriptlet>

List breadcrumbs = HPWebModel.getBreadcrumbItems();

// Breadcrumb 1
MenuItem item = new MenuItem();
item.setTitle("Breadcrumb 1");
item.setUrl("http://www.hp.com");
breadcrumbs.add(item);

// Breadcrumb 2
item = new MenuItem();
item.setTitle("Breadcrumb 2");
item.setUrl("http://www.hp.com");
breadcrumbs.add(item);

</jsp:scriptlet>

Note that the only properties of the MenuItem class that have any impact on the rendering of the breadcrumbs are the title and url properties. 

Applies To: Enhanced Layout
Message Key (Component): none
HPWebModel Property: {@link com.hp.frameworks.wpa.portal.hpweb.HPWebModel#getBreadcrumbItems()}

7.39 Generate Breadcrumb Items

7.43 Left Menu

The menu items displayed within the left navigation menu can be specified in one of two different ways: 1) the default, allow the HPWeb layout to generate the menu as configured from the VAP Site Console, or 2) programmatically using the leftMenuItems property. If the leftMenuItems property is set then it will be used.

In programmatically setting the menu, the leftMenuItems property accepts a list of {@link com.hp.frameworks.wpa.hpweb.MenuItem} objects.  Note that only objects of type ({@link com.hp.frameworks.wpa.hpweb.MenuItem}) should be added to the list.

The following example shows how to programmatically configure a sample menu.

<jsp:useBean id="HPWebModel" scope="request"
    class="com.hp.frameworks.wpa.portal.hpweb.HPWebModel" />

<jsp:scriptlet>

List leftMenu = HPWebModel.getLeftMenuItems();

// Non-clickable menu text
MenuItem item = new MenuItem();
item.setTitle("Heading 1");
leftMenu.add(item);

// Nested menu link
MenuItem child = new MenuItem();
child.setTitle("Link 1");
child.setUrl("http://www.hp.com");
item.getSubMenuItems().add(child);

// Separator
leftMenu.add(new MenuItem());

// Top-level menu link
item = new MenuItem();
item.setTitle("Link 2");
item.setUrl("http://www.hp.com");
leftMenu.add(item);

</jsp:scriptlet>

7.53 Meta Elements

This property will add additional <meta {type}="{typeValue}" content="{content}"> elements to the <head> element. {type} should be one of: name, http-equiv, scheme. {typeValue} should be appropriate for the given {type}, e.g. "keywords" can be used for type "name", "content-type" can be used for type "http-equiv", etc...  Retrieve the java.util.List<MetaTag> object and use the add(new {@link com.hp.frameworks.wpa.portal.hpweb.MetaTag}(type, typeValue, content)) method to include additional <meta> elements.

Applies To: Enhanced Layout
Message Key (Component): none
HPWebModel Property: {@link com.hp.frameworks.wpa.portal.hpweb.HPWebModel#getMetaTags()}

The following meta elements are included on each page:

  • <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
  • <meta http-equiv="Content-Style-Type" content="text/css">
  • <meta http-equiv="Cache-Control" content="no-cache">
  • <meta name="target_country" content="${languageTag}">
    • See the Locale section.
  • <meta name="hp_design_version" content="hpweb.1.2j">

Note that {@link com.hp.frameworks.wpa.portal.hpweb.HPWebModel#getMetaInfos()} has been deprecated. All the properties added to it continue to be rendered as <meta name="{propertyName}" content="{propertyValue}"> elements.

8. Misc

8.1 Locale

HPWeb layout will access the locale for the current user from one of the following sources and in the following order: authenticated user or guest user, current site default locale, default locale, or null object.

The {@link java.util.Locale} object property will be used to help perform the following functions:

  • Set the language attribute value (RFC 3066 language tag) in the <html> element.
  • Add a <meta name="target_country" contents="${languageTag}"> element in the <head> element.
  • Retrieve messages used by the HPWeb layout in the current locale.

8.2 Search Engine Indexing

Search engine indexing is enabled using start and stop indexing tags around the page content, and the page title, breadcrumbs, and tagline.
