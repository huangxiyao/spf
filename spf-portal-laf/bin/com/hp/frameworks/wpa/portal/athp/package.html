1. Overview

Portal applications running on the Vignette Application Portal (VAP) product that require the look-and-feel of the @hp UI standards can make use of the @hp VAP components.  Using the @hp layout allows developers to focus solely on rendering the main content portion of the page without having to worry about the details of standard header, footer and navigation menu (commonly referred to as the 'c-frame'). For more information about the @hp standard see the @hp Get Connected site.

Everything outlined in red in the picture below will be rendered by the @hp layout.  Everything NOT outlined in red is referred to as the "page content".

The @hp layout is supported by the following components to render the layout and various @hp design elements in the layout:

  • Theme component
    • @hp Theme
  • Grid components
    • @hp Layout Horizontal Navigation Grid
    • @hp Layout Split Navigation Grid
    • @hp Layout System Grid
  • Style type component
    • @hp Layout Config Style Type
  • Style components
    • @hp Layout Header Style
    • @hp Layout Footer Style
    • @hp Layout Horizontal Navigation Style
    • @hp Layout Vertical Navigation Style
    • @hp Layout Config Style
See the @hp Components section for more information.

2. System Requirements

The @hp components runs on Vignette Application Portal (VAP) product version 7.4 and above. Java Servlet 2.4, JSP 2.0 and JSTL 1.1 are required.

3. Installation

The @hp layout components are packaged in a AtHPComponents-<version>.car file is downloadable from the WPA @hp VAP Components Downloads directory.   Download the latest version to a local temporary directory.  

Use the VAP Server Console to install/import the components.   After they are imported, share each @hp component to the site(s) where they will be used; otherwise, they won't be accessible. Since the @hp Components contain class JAR files the application server has to be restarted twice to load the classes.

3.1 Vignette Page Display Issue

There is a known Vignette issue under certain conditions with the default Page Display secondary page and it affects the @hp layout horizontal menu display.

To fix this problem perform the following to use the standard Page Display secondary page:

  1. Go to Server Console -> Components -> Pages -> Secondary Page List . On the page, click on the "Page Display" secondary page checkbox, select "Share to..." from the combo box and then the "Do It" button. Then select the site(s) that you want to share to.
  2. Go to the Site Console for that site -> Site Settings -> Appearance -> Secondary Pages ->. For the "Page Display" Page Type ensure you select "Page Display" in the Use Secondary Page column.

4. Usage

Use the VAP Admin Console to apply the @hp layout to a site or a set of pages by setting its theme to @hp Theme and its grid to one of the @hp grids. You can customize the @hp layout to your portal and pages through component properties or programmatically. Component properties customization involves setting layout properties in the @hp Components while programmatic customization involves setting layout properties to the AtHPModel named request-scoped bean. See the Layout Customization section for more information.

5. @hp Components

5.1 Style Types

All of the page elements in the @hp c-frame match with the style types provided by VAP. The @hp components include implementations for each of the following VAP style types:

  • Header
  • Footer
  • Horizontal Navigation
  • Vertical Navigation
  • HP Layout Config
The HP Layout Config Style Type is provided for a style instance, e.g. @hp Layout Config Style, to hold custom code for programmatic layout configuration.

5.2 Themes

@hp components include the @hp Theme that aggregates the following styles:

  • @hp Header Style
  • @hp Footer Style
  • @hp Vertical Navigation Style
  • @hp Horizontal Navigation Style
  • @hp Layout Config Style

5.3 Grids

Three @hp grids are provided:

  • @hp System Grid - This grid is used for system pages and pages that do not require any navigation. Neither the top horizontal navigation nor the left vertical navigation is displayed on this grid.
  • @hp Horizontal Navigation Grid - This grid is used for pages that require a top horizontal navigation but no left vertical navigation.
  • @hp Split Navigation Grid - This grid is used for pages that require a top horizontal navigation and a left vertical navigation.

5.4 Styles

5.4.1 @hp Header Style

The @hp Header Style displays:

  • @hp standard banner
  • Page title, welcome message, warning message
  • Breadcrumbs

5.4.2 @hp Footer Style

The @hp Footer Style displays:

  • Privacy link, terms and conditions link, feedback link, support link, etc

5.4.3 @hp Horizontal Navigation Style

The @hp Horizontal Navigation Style displays the the first and second levels navigation items as configured in the Vignette Admin Console in the top horizontal navigation menu. Per @hp web standards, a maximum of two levels is displayed. The navigation split level, if set in the Vignette Admin Console, is ignored.

5.4.4 @hp Vertical Navigation Style

The @hp Vertical Navigation Style displays up to two levels of navigation per @hp web standards. The default behavior is to display the third and fourth navigation levels for the current page as configured in the Vignette Admin Console. If the current page is a first level navigation item, then the left menu will show its second and third level navigation items. The left menu can be created programmatically during run-time, overriding the default behavior. See the Left Menu configuration property for more information.

5.4.5 @hp Layout Config Style

The @hp Layout Config Style is the style where to include code for customizing the @hp layout. It must not contain HTML that outputs HTTP responses. Each @hp grid includes this style at the beginning of the grid's execution, so that the layout properties may be initialized before the @hp layout accesses them for rendering.

The custom code goes into the primary JSP file, athp_layout_config.jsp. Also included is the sample_athp_layout_config.jsp file which contains sample customizations. See the next section for more information on layout customization.

Additionally, customized code for including in the <head> element can be placed in the athp_layout_config_head.jsp file. Code such as javascript code and CSS styles can be placed in this file.

6. Layout Customization

A good portion of the markup rendered by the @hp layout is static HTML, however, there are a number of design elements which are typically customized on a site-by-site or even page-by-page basis. One item that is most typically customized is the page title.

For most of the configurable page elements, there are two different mechanisms available for specifying a custom value: 1) a mechanism utilizing @hp component properties, and 2) a programmatic mechanism utilizing the AtHPModel request-scoped attribute object. A property with a non-null value in the AtHPModel object will take precedence over a value in the component property. It is up to the application developer to choose which approach is preferred. The general rule is to use component properties for static constant values and the programmatic mechanism for dynamic run-time values. It is not necessary to set values for all of the properties. Some properties can be enabled to auto-generate values at run-time, e.g. Left Menu property and Breadcrumbs property.

6.1 Customization in Component Properties

This approach involves setting property values in the @hp components. See Configuration Reference for the list of properties and components.

6.2 Programmatic Customization

The com.hp.frameworks.wpa.portal.athp.AtHPModel bean type contains properties which are customizations to the @hp layout. The programmatic approach involves setting the property values on the AtHPModel named request-scoped attribute bean of that type. Each @hp grid provides an integration point by including the @hp Layout Config Style. The athp_layout_config.jsp primary file in the @hp Layout Config Style is where a portal can place its layout customization code.

The following sample code references the AtHPModel bean, and set @hp properties on it.

<jsp:useBean id="AtHPModel" scope="request" class="com.hp.frameworks.wpa.portal.athp.AtHPModel" />

<jsp:setProperty name="AtHPModel" property="title" value="My Page Title" />
<jsp:setProperty name="AtHPModel" property="feedbackUrl" value="http://myfeedback.hp.com" />

7. Configuration Reference

The following sections describe all of the configurable page elements along with the corresponding property names in component properties files and AtHPModel object.

7.1 Body Style Disabled Flag

The boolean value of this property determines whether or not the standard @hp body content styles are disabled.  A value of "true" will prevent the header_body.css file from being automatically included.  If no value is supplied for this property, a value of "false" is assumed and the body styles will be enabled.

Message Key (Component): athp.bodyStyleDisabled (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setBodyStyleDisabled(boolean)}

7.2 Title

The value of this property will be rendered as the page title (sometimes referred to as the page header or banner).  Note that this is different than the browser window title, which is set using the Window Title property. If no title string is provided, the default title is the browser window title.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setTitle(String)}

7.3 Secondary Title

The value of this property will be rendered as the secondary page title rendered immediately after the value of the title property.  This is an optional page element.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setSecondaryTitle(String)}

7.4 Tertiary Title

The value of this property will be rendered as the tertiary page title rendered immediately after the value of the secondaryTitle property.  Note that the value of this property will only be used if a value is also provided for the secondaryTitle property.  This is an optional page element.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setTertiaryTitle(String)}

7.5 Greeting Message

The value of this property will be rendered as the greeting message displayed beneath the top banner.  This is an optional page element.  If no value is specified, no greeting message will be rendered.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setGreetingMessage(String)}

7.6 Warning Message

The value of this property will be rendered as the warning message displayed beneath the top banner.  This is an optional page element.  If no value is specified, no warning message will be rendered.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setWarningMessage(String)}

7.7 Login URL

The value of this property will be used as the destination URL for the "Log On" hyperlink.  This is an optional page element.  If not value is specified, the "Log On" hyperlink will not be displayed.

Message Key (Component): athp.loginUrl (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setLoginUrl(String)}

7.8 Logout URL

The value of this property will be used as the destination URL for the page the user is directed to after logging out via the "Log Off" button.  This is an optional page element.  If a value is specified for this property, a value must also be specified for the loginUrl property.

Message Key (Component): athp.logoutUrl (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setLogoutUrl(String)}

7.9 Site Search ID

The value of this property will be used as the search ID to use when searching the local site.  This is an optional page element.  Setting a value for the site search ID will cause radio buttons to be displayed beneath the search box in the top banner that will allow the user to select between searching the entire intranet or just the local site.

Message Key (Component): athp.siteSearchID (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setSiteSearchID(String)}

7.10 Site Search Label

The value of this property will be used as the text for the site search label. This text is displayed next to the site search radio button within the search box.  If a value is specified for this property, a value must also be specified for the siteSearchID property.

Message Key (Component): athp.siteSearchLabel (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setSiteSearchLabel(String)}

7.11 My Links Disabled Flag

The boolean value of this property is used to indicate whether or not the "My Links" hyperlink should be displayed as part of the banner.  A value of "true" indicates that the "My Links" hyperlink should not be displayed, while a value of "false" (the default) indicates that the "My Links" hyperlink should be displayed.

Message Key (Component): athp.myLinksDisabled (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setMyLinksDisabled(boolean)}

7.12 Breadcrumbs

Breadcrumbs are used to represent the current page's location in the site hierarchy and are displayed immediately beneath the top navigation menu.  Breadcrumbs are automatically generated by the @hp Horizontal Navigation component if the Generate Breadcrumbs property is to true.   It also can be set by retrieving its List property and adding breadcrumb strings to it (sample code below).

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#getBreadcrumbItems()}

<jsp:useBean id="AtHPModel" scope="request"
    class="com.hp.frameworks.wpa.portal.athp.AtHPModel" />

<jsp:scriptlet>

List breadcrumbs = AtHPModel.getBreadcrumbItems();

// Breadcrumb 1
breadcrumbs.add("Breadcrumb1");

// Breadcrumb 2
breadcrumbs.add("Breadcrumb2");

</jsp:scriptlet>

7.13 Generate Breadcrumbs

The value of this boolean property will generate the default breadcrumb items for rendering if the value is set to true. The list of breadcrumb items consists of parent navigation items along with the current page item. If no value is supplied for this property no breadcrumb item is generated.

Message Key (Component): athp.generateBreadcrumbs (AtHP Header)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setGenerateBreadcrumbs(boolean)}

7.14 Horizontal Menu Style Flag

The boolean value of this property indicates whether or not the 2nd-level of the top navigation menu should be rendered horizontally.  A value of "true" indicates that the 2nd-level menu items should be rendered as a horizontal menu while a value of "false" (the default) indicates that the 2nd-level menu items should be rendered as a vertical menu.

Message Key (Component): athp.horizontalMenuStyle (AtHP Horizontal Navigation)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setHorizontalMenuStyle(boolean)}

7.15 Top Menu

The menu items displayed within the top (horizontal) navigation menu is constructed from the navigation tree of the portal created from the VAP Admin Console.   There is no programmatic customization.    Per @hp web standards, at most two levels of menu are displayed.  A menu item is either a link or separator (VAP Spacer item type, 2nd level only).  If a top-level menu item has sub-menu items but the user does not have permission to access any of the items,  then the top-level menu item is not shown.

7.16 Left Menu

The menu items displayed in the left vertical navigation can be specified in one of two different ways: 1) the default, allow the @hp layout to generate the menu as configured from the VAP Admin Console, or 2) programmatically using the leftMenuItems property. If the leftMenuItems property is set then it will be used.

The menu items displayed is either a link, label, separator, or spacer type. Per @hp web standards, at most two levels are displayed.

In programmatically setting the menu, the leftMenuItems property accepts a list of {@link com.hp.frameworks.wpa.portal.athp.MenuItem} objects. Note that only objects of type {@link com.hp.frameworks.wpa.portal.athp.MenuItem} should be added to the list.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#getLeftMenuItems()}

The following example shows how to programmatically configure a sample menu.

<jsp:useBean id="AtHPModel" scope="request"
    class="com.hp.frameworks.wpa.portal.athp.AtHPModel" />

<jsp:scriptlet>

List leftMenu = AtHPModel.getLeftMenuItems();

// Label menu item
MenuItem item = new MenuItem("id1", "text item 1");
leftMenu.add(item);

// Separator menu item
item = new MenuItem();
item.setSpacer("true");
leftMenu.add(item);

// Link menu items
item = new MenuItem("id2", "hp support", "http://support.hp.com", "_new");
leftMenu.add(item);
item = new MenuItem("id3", "hp shopping", "http://shopping.hp.com", "_new");
leftMenu.add(item);

// Separator menu item
item = new MenuItem();
item.setSeparator("true");
leftMenu.add(item);

// Label menu item with submenu
item = new MenuItem("id4", "external sites");
leftMenu.add(item);

MenuItem child = new MenuItem("id4_1", "espn", "http://espn.com", "_new");
item.getSubMenuItems().add(child);
child = new MenuItem("id4_2", "cnn", "http://cnn.com", "_new");
item.getSubMenuItems().add(child);

</jsp:scriptlet>

In the example above, the menu items have been specified as follows:

  • A non-clickable menu item with id "id1" displaying the text "text item 1"
  • A menu spacer
  • A clickable menu item with id "id2" displaying the text "hp support" that opens to a new browser window with target "_new"
  • A clickable menu item with id "id3" displaying the text "hp shopping" that opens to a new browser window with target "_new"
  • A menu separator
  • A non-clickable menu item label with "id4" displaying the text "external sites" that has an expanded sub-menu
  • A clickable sub-menu item with parent id "id4" and id "id4_1" displaying the text "espn" that opens to a new browser window with target "_new"
  • A clickable sub-menu item with parent id "id4" and id "id4_2" displaying the text "cnn" that opens to a new browser window with target "_new"

7.17 Feedback URL

The value of this property is used as the destination URL for the feedback link in the page footer.  If no value is supplied for this property the feedback hyperlink will point to the default feedback page.

Message Key (Component): athp.feedbackUrl (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setFeedbackUrl(String)}

7.18 Support URL

The value of this property is used as the destination URL for the support link in the page footer.  If no value is supplied for this property the support hyperlink will point to the default support page.

Message Key (Component): athp.supportUrl (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setSupportUrl(String)}

7.19 Target Frame

The value of this property is used as the name of the window/frame in which the feedback and support URLs will be opened.  If no value is specified for this property the feedback and support pages will be opened in the current window/frame.

Message Key (Component): athp.targetFrame (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setTargetFrame(String)}

7.20 Confidential Label Flag

The boolean value of this property is used to determine whether or not the "HP Confidential" label is rendered as part of the page footer.  A value of "true" indicates that the "HP Confidential" label should be rendered.  If neither the confidentialLabel or publicLabel properties are set to "true", the default "HP Restricted" label will be rendered.

Message Key (Component): athp.confidentialLabel (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setConfidentialLabel(boolean)}

7.21 Public Label Flag

The boolean value of this property is used to determine whether or not the "HP Public" label is rendered as part of the page footer.  A value of "true" indicates that the "HP Public" label should be rendered.  If neither the confidentialLabel or publicLabel properties are set to "true", the default "HP Restricted" label will be rendered.

Message Key (Component): athp.publicLabel (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setPublicLabel(boolean)}

7.22 Revision Date Displayed Flag

The boolean value of this property is used to determine whether or not the page revision date should be rendered as part of the page footer.  A value of "true" indicates that the page revision date should be displayed while a value of "false" (the default) indicates that the page revision date should not be displayed.

Message Key (Component): athp.revisionDateDisplayed (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setRevisionDateDisplayed(boolean)}

7.23 Version Number

The value of this property is displayed as the page version number in the page footer.  This is an optional page element.  If not value is specified for this property, not version string will be displayed.

Message Key (Component): athp.versionNumber (AtHP Footer)
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setVersionNumber(String)}

7.24 Meta Elements

This property will add additional <meta name="<name>" contents="<value>"> elements to the <head> element.  Retrieve the java.util.Properties object and use the setProperty(name, value) method to include additional <meta> elements.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#getMetaInfos()}

<jsp:useBean id="AtHPModel" scope="request"
    class="com.hp.frameworks.wpa.portal.athp.AtHPModel" />

<jsp:scriptlet>

Properties metaInfos = AtHPModel.getMetaInfos();
metaInfos.setProperty("robots", "noindex,nofollow");
metaInfos.setProperty("country", "us");

</jsp:scriptlet>

7.25 Window Title

The value of this property will be rendered as the browser window title which is displayed in the title bar of the user's browser. If no value is specified for this property the Vignette JSP custom tag <vgn-portal:pageContentTitle /> will be used to render the window title.

Message Key (Component): none
AtHPModel Property: {@link com.hp.frameworks.wpa.portal.athp.AtHPModel#setWindowTitle(String)}
